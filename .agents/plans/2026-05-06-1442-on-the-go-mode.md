## Goal

Build a mobile-first, voice-first mode that lets users drive their T3 Code coding agent hands-free from their phone via a notifications panel, a TTS summary bot, a back-and-forth voice conversation, and an explicit commit gesture that injects an optimized prompt back into the main thread.

## Acceptance criteria

The branch is feature-complete when:

- [ ] `/on-the-go` route works on mobile Safari and mobile Chrome via the existing T3 Code pairing flow.
- [ ] Inbox surfaces threads with status `awaiting` or `errored`; deduped per thread; errored floats to top.
- [ ] Tapping a card initiates a hands-free voice flow with TTS summary, auto-listen turn-taking, and interrupt support.
- [ ] Saying "ship it" (or tapping the button) triggers a 3-second cancel countdown with audible + visible preview of the optimized prompt; the prompt is delivered to the main thread on countdown expiry; the notification clears.
- [ ] Cancel during countdown returns to conversation with full history intact.
- [ ] Pause snapshots full session state; resuming restores the conversation with a TTS context-restore prompt.
- [ ] Idle timeout (30s + 15s "still there?") auto-pauses to the Paused tab.
- [ ] All three `SummaryAdapter` implementations (OpenAI, Anthropic, MainAgentCli) are functional and selectable in Settings.
- [ ] Mic permission denied → text-input fallback works.
- [ ] `composePrompt` failure → envelope-wrapped verbatim transcript fallback works.
- [ ] Browser `Notification` API fires system banners when the tab is loaded but hidden.
- [ ] Onboarding runs once on first visit; settings expose all toggles afterwards.
- [ ] FSM tests + store tests pass with 100% branch coverage.
- [ ] One end-to-end happy-path Playwright test passes with fake adapters.
- [ ] Manual device verification on iOS Safari + Android Chrome documented and signed off.

## Notes

**Spec:** `docs/superpowers/specs/2026-05-06-on-the-go-mode-design.md` (the contract).

**Architecture (one-line):** Isolated feature folder at `apps/web/src/onTheGo/` with two swappable interfaces (`VoiceAdapter`, `SummaryAdapter`), a hand-rolled state-machine orchestrator, two stores, and mobile-first variants of existing shadcn primitives. Reuses existing T3 Code RPC for thread state and pairing for auth — **no server changes for v1**.

**Deliberately deferred (v1.1+):**
- Web Push for OS-level notifications when the app is fully closed (needs PWA + service worker + VAPID).
- Cloud `VoiceAdapter` impls (OpenAI Realtime, ElevenLabs+Whisper).
- "Sent" history view of recently committed prompts.
- Configurable prompt-style preference for the optimizer skill file.
- Multi-device session coordination.

**Decisions locked during brainstorming** (not up for relitigation during implementation without a strong reason):
- Browser-native voice as the v1 default; cloud as later upgrade behind the same interface.
- Pluggable `SummaryAdapter` (OpenAI default, Anthropic + MainAgentCli also v1).
- Auto-listen turn-taking with explicit `"ship it"` commit gate + 3s cancel countdown.
- Fallback to envelope-wrapped verbatim transcript when `composePrompt` fails.
- Two tabs only (Inbox + Paused); no Sent tab in v1.
- `Notification` API for off-app awareness (not Web Push); PWA install deferred.
- Card-based notifications panel (~180-200px tall); errored floats to top.
- Three huge buttons at the bottom of the voice screen (Pause / Mic / Ship it), ≥64pt tall.
- Phone-call style voice screen with swipe-up transcript drawer.
- Theming = existing zinc / `base-mira` palette, just larger sizing tokens.
